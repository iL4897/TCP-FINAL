# 멀티스레드 기반 끝말잇기 게임 (Python)

## 개요
#### 프로젝트 때 챗봇 프레임워크(C#)를 활용해 개발했던 1:1 채팅 방식의 끝말잇기를
#### TCP 연결을 통한 다중 접속 채팅 방식으로 개선.

## 목적
#### 멀티 쓰레드 개념 이해.
#### 챗봇 프레임워크는 사용자와 컴퓨터 간의 1:1 접속이었지만,
#### 멀티 스레드를 사용함으로써 접속하는 모든 사용자와 같이 플레이할 수 있는 채팅 형식의 끝말잇기 게임 개발.
#### 카카오톡 등과 같은 메신저 프로그램처럼 사용자의 구분과 채팅방에서 갖춰야할 기본 기능들의 모든 구현.

## 설계
##### 클라이언트와 서버는 정보를 보내는 동시에 수신할 수 있게끔 구현한다.
##### 큐를 사용하여 새로운 클라이언트의 접속 여부와 클라이언트들의 정보를 담는다.
##### 국립 국어원에서 제공하는 API 정보를 활용해서 사전 검색 기능 구현한다.
##### 서버와 접속이 성공시 닉네임 값을 처음으로 넘겨 받도록 한다.
##### 닉네임과 클라이언트 연결 정보를 리스트에 담아 같은 인덱스로 구성한다. (생성 및 삭제시 해당 접속 정보의 인덱스를 활용)

## 기능
#### 강의자료에 있는 UI를 바탕으로 카카오톡과 최대한 비슷하게 UI 설계
#### 채팅방에 입장, 퇴장시 메세지 출력
#### 클라이언트별 닉네임 지정
#### 끝말잇기 게임 중 단어 중복, 없는 단어 등의 예외 처리
#### 게임이 진행중에 입장 시 마지막으로 사용한 단어를 안내
#### 클라이언트 연결과 함께 소켓 생성 및 정보 입력
#### 클라이언트가 채팅방을 나가면 종료와 함께 소켓 및 정보 삭제


## 문제점
#### 1. Tkinter 창을 두 개로 구성해 닉네임 받는 창과 채팅방을 따로 구성하고 싶었으나 서버와 연결은 문제없지만 클라이언트 간의 소통이 불가하다는 문제점.
###    하나의 Tkinter 창을 활용하여 화면에 구성하게끔 설정값만 수정해서 코드 작성 (부분 해결)
#### 2. 클라이언트가 '접속 - 닉네임 입력' 순으로 실행될 때는 오류가 없지만 '접속 - 대기 - 다른 클라이언트 접속' 순으로 실행될 때 해당 클라이언트가 무시되는 문제점.
